# 六爻迭代解卦 Skill

## 核心边界

本 skill 用于六爻/纳甲解卦辅助、古籍检索、案例复盘和规则迭代。它不是事实预测器，也不应用于医疗、法律、金融等高风险决策。

必须遵守：

1. 新卦默认使用 `fresh` 模式。
2. `fresh` 模式不得读取 `memory/casebook.jsonl`、`memory/feedback_log.jsonl` 或 `memory/incoming_cases/`。
3. 历史案例只能在用户明确要求“参考历史类案、复盘、学习、总结旧案”时调用。
4. 古籍文本可作为参考依据，但必须和当前卦盘结构分开表述。
5. 单个案例反馈不得直接改写成硬规则。
6. 输出应使用概率性、条件性语言，不做绝对断言。

## 运行模式

### fresh：默认新卦模式

触发词包括：`解卦`、`六爻`、`纳甲`、`按 fresh 模式`、`不要参考历史案例`。

执行规则：

- 只分析当前卦盘与当前问题。
- 可以调用 `scripts/retrieve_passages.py` 或 `scripts/retrieve_for_divination.py` 检索古籍。
- 不得调用 `scripts/list_cases.py`、`scripts/summarize_learning.py` 或读取 `memory/casebook.jsonl`。
- 结尾必须声明：`本次为 fresh 模式，未调用历史案例。`

### literature_only：只查古籍

用于学习规则、检索断语、比较书籍，不输出具体预测。

### analogical_case：类案参考模式

只有当用户明确要求“参考旧案/类案”时使用。必须：

1. 先给出当前卦的独立判断。
2. 再列出类案。
3. 分开说明相似点和差异点。
4. 不允许用旧案覆盖当前卦盘证据。

### posthoc_learning：反馈复盘模式

用于事情发生后的反馈记录和学习。必须把反馈分为：

- 支持原判断
- 部分支持
- 反向结果
- 无法验证
- 样本异常

反馈只能形成“待验证假设”，不能自动成为规则。

## 输入要求

优先要求用户提供完整排盘：

- 问题原文
- 起卦时间
- 起卦方式：铜钱、数字、时间、手动、外部排盘
- 本卦、变卦
- 动爻
- 世应
- 六亲
- 六神
- 日辰、月建、旬空
- 问题背景和已知事实

如果信息不全：

- 可以先基于已有信息给出“不完整分析”；
- 不要自行编造日辰、月建、旬空、世应、六亲；
- 可提示用户补充外部排盘结果。

## 解卦流程

每次 fresh 解卦按以下顺序：

1. **确认占事类型**：婚恋、财运、工作、考试、疾病、行人、失物、合作等。
2. **确定用神**：说明用神选择理由，必要时列辅助用神。
3. **看世应关系**：世代表求测方，应代表对方/环境/事情对象。
4. **看日月旺衰**：用神、世爻、应爻、关键动爻是否得月日生扶、冲克、空破。
5. **看动爻链条**：动爻生克冲合谁，是否化进、化退、化回头生/克、化空破墓绝。
6. **看六亲结构**：父母、兄弟、子孙、妻财、官鬼的主题含义。
7. **看六神取象**：只作为象意辅助，不得压过五行生克和用神旺衰。
8. **看伏神飞神**：若盘中有伏神，说明飞伏关系；若没有信息，不要编造。
9. **综合判断**：分为主判断、支持证据、反向证据、条件限制。
10. **给出应期线索**：只能在证据足够时给出，必须标注不确定性。

## 古籍检索策略

古籍 md/txt 放在：

- `resources/books_raw_md/`：原始 OCR 或 MinerU Markdown。
- `resources/books_corrected/`：清洗校对后的文本，优先用于索引。

建索引：

```bash
python scripts/ingest_books.py
```

检索：

```bash
python scripts/retrieve_passages.py --query "用神 世爻 应爻 动爻 空亡 月建 日辰" --top-k 8
```

组合检索：

```bash
python scripts/retrieve_for_divination.py --base "婚姻 官鬼 妻财 世应 六合 冲克" --top-k 5
```

检索结果使用规则：

- 优先用核心规则书，如《增删卜易》《卜筮正宗》《易隐》《黄金策》。
- 汇编类、歌诀类用于旁证。
- 不能只因为某条断语相似就忽略当前卦盘的用神旺衰。

## 案例与反馈

只有在用户要求保存或复盘时才写入 memory。

添加案例：

```bash
python scripts/add_case.py --file examples/sample_case.json
```

添加反馈：

```bash
python scripts/update_feedback.py --case-id CASE_ID --outcome partial --note "实际结果..."
```

清空运行记忆：

```bash
python scripts/reset_runtime_memory.py --backup
```

## 禁止行为

- 禁止在 fresh 模式读取旧案例。
- 禁止把用户隐私案例写入公开 README 或 SKILL.md。
- 禁止将未校对 OCR 直接作为高权重依据。
- 禁止把古籍断语当成不需要条件的绝对结论。
- 禁止为了迎合用户而只列支持证据，不列反向证据。

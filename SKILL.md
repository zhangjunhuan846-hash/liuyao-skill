# 六爻迭代解卦与流派冲突并列分析 Skill

## 0. 核心定位

本 skill 用于六爻/纳甲解卦辅助、古籍检索、流派冲突展开、案例复盘和规则迭代。它不是事实预测器，也不应用于医疗、法律、金融等高风险决策。

本 skill 的核心目标不是把所有六爻流派揉成一个答案，而是：

> 每次解卦先按当前卦盘独立建立证据链；若古籍、注本、现代案例或不同流派出现矛盾，必须列出所有可辨识结果，并标注流派、来源、判断、证据链、冲突点和置信度。

必须遵守：

1. 新卦默认使用 `fresh` 模式。
2. `fresh` 模式不得读取 `memory/casebook.jsonl`、`memory/feedback_log.jsonl` 或 `memory/incoming_cases/`。
3. 历史案例只能在用户明确要求“参考历史类案、复盘、学习、总结旧案”时调用。
4. 古籍文本可作为参考依据，但必须和当前卦盘结构分开表述。
5. 单个案例反馈不得直接改写成硬规则。
6. 输出应使用概率性、条件性语言，不做绝对断言。
7. 遇到流派矛盾时不得强行融合，必须并列输出。

## 1. 默认主体系

用户未指定流派时，默认采用：

> 传统纳甲六爻主线，以《火珠林》《黄金策》《增删卜易》《卜筮正宗》一系规则为主。

默认分析顺序：

1. 明确占事与问题边界。
2. 定主用神、辅用神、世应。
3. 看月建、日辰、旬空、旺相休囚死。
4. 看动爻、变爻、回头生克、进退神。
5. 看原神、忌神、仇神、闲神。
6. 看冲、合、刑、害、墓、绝、破。
7. 看飞伏神；仅在用神不现或伏藏关键时提升权重。
8. 看六神、爻位、卦象、象意；只作补充，不得推翻主结构。
9. 看应期候选；不得强行唯一化。
10. 若存在流派冲突，进入“流派并列结果”模式。

## 2. 运行模式

### 2.1 fresh：默认新卦模式

触发词包括：`解卦`、`六爻`、`纳甲`、`按 fresh 模式`、`不要参考历史案例`。

执行规则：

- 只分析当前卦盘与当前问题。
- 可以调用 `scripts/retrieve_passages.py` 或 `scripts/retrieve_for_divination.py` 检索古籍和冲突表。
- 不得调用 `scripts/summarize_learning.py` 或读取 `memory/casebook.jsonl`、`memory/feedback_log.jsonl`。
- 若用户未指定流派，按默认传统纳甲六爻主体系先给主判断。
- 若发现流派冲突，必须并列列出。
- 结尾必须声明：`本次为 fresh 模式，未调用历史案例。`

### 2.2 literature_only：只查古籍 / 规则学习

用于学习规则、检索断语、比较书籍，不输出具体预测。适合检索：用神、世应、动变、飞伏神、六神象意、应期等。

### 2.3 school_comparison：流派比较模式

当用户要求“按不同流派都列出来”“有矛盾都标注”“参考南山真人/飞伏神/象法”等，使用本模式。

输出必须包含：

- 默认传统纳甲主线判断；
- 用户指定流派判断；
- 可辨识的辅助流派判断；
- 冲突点；
- 本 skill 默认采用的路径；
- 其他路径为何保留。

### 2.4 analogical_case：类案参考模式

只有当用户明确要求“参考旧案/类案/南山真人卦例/现代案例”时使用。必须：

1. 先给出当前卦的独立判断。
2. 再列出类案或现代案例派解释。
3. 分开说明相似点和差异点。
4. 不允许用旧案覆盖当前卦盘证据。
5. 将现代案例派结果标为 `低` 或 `中低` 置信度，除非它同时受到核心结构证据支持。

### 2.5 posthoc_learning：反馈复盘模式

用于事情发生后的反馈记录和学习。必须把反馈分为：

- 支持原判断
- 部分支持
- 反向结果
- 无法验证
- 样本异常

反馈只能形成“待验证假设”，不能自动成为规则。

## 3. 流派冲突处理硬规则

### 3.1 不强行融合

若不同流派、古籍、注本、现代案例之间出现冲突，不得揉成一个结论。必须按流派分别列出：

- 流派/体系名称
- 规则来源
- 该体系下的判断
- 证据链
- 与其他体系的冲突点
- 置信度

### 3.2 规则优先级

优先级只用于说明“默认采用哪一路径”，不得删除其他流派结果。

1. 用户指定流派。
2. 明确占事规则。
3. 默认传统纳甲六爻主体系。
4. 多部核心古籍共同支持。
5. 原文规则。
6. 后人注解。
7. 辅助系统：飞伏神、六神象意、神煞、纳音等。
8. 现代案例。
9. 杂项未校资料。

### 3.3 主判据与辅助判据

主判据：用神、世应、月日、动变、生克制化、原神忌神、冲合刑害、空破墓绝。

辅助判据：飞伏神、六神、神煞、纳音、爻位、卦象、现代卦例。

辅助判据可以补充细节，但不得单独推翻主结构。若两者相反，必须并列输出，并说明冲突来自哪里。

### 3.4 触发流派并列结果的情况

遇到下列情况必须进入“流派并列结果”模式：

- 用神取法不同。
- 世应与用神判断方向不同。
- 月日旺衰权重不同。
- 动变、回头生克、进退神解释不同。
- 飞伏神是否取用存在差异。
- 六神、神煞、爻位、纳音、象意与主结构相反。
- 应期取法不同。
- 古籍规则与现代案例判断不同。
- 用户同时要求参考多位作者或多种流派。

### 3.5 置信度标注

每条判断必须标注置信度：

- 高：用神、世应、月日、动变、生克制化至少三类核心证据同向。
- 中：两类核心证据同向，或核心证据与辅助象意基本一致。
- 低：主要来自六神、爻位、神煞、纳音、现代案例、单一口诀或象意扩展。
- 不足：卦信息缺失、时间缺失、起卦方式不清、文本来源不明，无法稳定判断。

## 4. 输入要求

优先要求用户提供完整排盘：

- 问题原文和问题背景；
- 希望判断的时间范围；
- 起卦时间：公历/农历、时辰、所在地或时区；
- 起卦方式：手摇、铜钱、数字、时间、软件排盘等；
- 本卦、变卦、动爻；
- 世应；
- 六亲；
- 六神；
- 纳甲地支；
- 日辰、月建、旬空；
- 是否指定流派：默认、增删卜易、卜筮正宗、飞伏神、六神象意、南山真人等。

如果信息不全：

- 可以先基于已有信息给出“不完整分析”；
- 不要自行编造日辰、月建、旬空、世应、六亲；
- 必须说明缺失项对判断的影响；
- 不得输出高置信度结论。

## 5. fresh 解卦流程

每次 fresh 解卦按以下顺序：

1. **卦盘完整性检查**：列已知项、缺失项、可靠性。
2. **确认占事类型**：婚恋、财运、工作、考试、疾病、行人、失物、合作等。
3. **确定用神**：说明主用神、辅用神和取用理由；若有冲突，列出不同取法。
4. **看世应关系**：世代表求测方，应代表对方/环境/事情对象。
5. **看日月旺衰**：用神、世爻、应爻、关键动爻是否得月日生扶、冲克、空破。
6. **看动爻链条**：动爻生克冲合谁，是否化进、化退、化回头生/克、化空破墓绝。
7. **看六亲结构**：父母、兄弟、子孙、妻财、官鬼的主题含义。
8. **看飞伏神**：若盘中有伏神，说明飞伏关系；若无信息，不要编造。
9. **看六神取象**：只作为象意辅助，不得压过五行生克和用神旺衰。
10. **流派冲突检查**：检查用神、旺衰、飞伏、六神、应期是否存在不同路径。
11. **综合判断**：分主判断、并列判断、支持证据、反向证据、条件限制。
12. **给出应期候选**：只能作为候选集合，必须标注不确定性。

## 7. 古籍与知识库放置规则

原始 PDF 放在：

- `raw_pdfs/`：原始 PDF，不直接用于检索。

原始 OCR / MinerU / Markdown 输出，先放：

- `resources/books_raw_md/`

清洗校对后的 md/txt，放：

- `resources/books_corrected/`

按流派分层后的知识库，放：

- `knowledge/00_basic_terms/`：六亲、世应、用神、月建日辰、动变、空亡冲合等。
- `knowledge/01_core_classics/`：火珠林、黄金策、增删卜易、卜筮正宗。
- `knowledge/02_auxiliary_systems/`：易隐、断易天机、飞伏神、六神象意、神煞、纳音等。
- `knowledge/03_modern_cases/`：南山真人卦例、现代案例、论坛卦例、课程讲义；默认不检索。
- `knowledge/04_conflict_registry/`：用神冲突表、旺衰冲突表、飞伏神冲突表、六神象意冲突表、应期冲突表。

建议流程：

```text
PDF/OCR 原文 → books_raw_md/ → 清洗校对 → books_corrected/ 或 knowledge/分层目录 → ingest_books.py → resources/index/book_chunks.jsonl
```

## 8. 元数据规则

每个 md 文件开头建议加 YAML frontmatter。

核心古籍示例：

```yaml
---
title: 火珠林
school: 传统纳甲六爻
source_type: core_classic
authority_level: A
retrieval_default: true
use_mode: rule_source
notes: 作为默认主体系基础规则来源之一。
---
```

现代案例示例：

```yaml
---
title: 南山真人卦例
school: 现代案例派
source_type: modern_case
authority_level: C
retrieval_default: false
use_mode: example_only
warning: 仅用于参考象意展开，不作为基础规则来源，不得直接套断。
---
```

## 9. 检索命令

建索引：

```bash
python scripts/ingest_books.py
```

默认检索核心规则与冲突表，不包含现代案例和杂项未校资料：

```bash
python scripts/retrieve_passages.py --query "用神 世爻 应爻 动爻 空亡 月建 日辰" --top-k 8
```

需要检索现代案例时显式开启：

```bash
python scripts/retrieve_passages.py --query "南山真人 感情 世应 六神" --include-modern-cases --top-k 8
```

组合检索：

```bash
python scripts/retrieve_for_divination.py --base "婚姻 官鬼 妻财 世应 六合 冲克" --top-k 5
```

组合检索并包含现代案例：

```bash
python scripts/retrieve_for_divination.py --base "婚姻 官鬼 妻财 世应 六合 冲克" --include-modern-cases --top-k 5
```

检索结果使用规则：

- 优先使用核心规则书：《火珠林》《黄金策》《增删卜易》《卜筮正宗》。
- 辅助流派可用于并列解释，但须标注流派。
- 现代案例只能在用户明确要求时调用。
- OCR 未校对文本降权。
- 一条断语只能作为证据之一，不能替代当前卦盘结构。

## 10. 案例与反馈

只有在用户要求保存或复盘时才写入 memory。

添加案例：

```bash
python scripts/add_case.py --file examples/sample_case.json
```

添加反馈：

```bash
python scripts/update_feedback.py --case-id CASE_ID --outcome partial --note "实际结果..."
```

汇总学习：

```bash
python scripts/summarize_learning.py
```

清空运行记忆：

```bash
python scripts/reset_runtime_memory.py --backup
```

## 11. 对南山真人等现代卦例的处理

南山真人、现代论坛卦例、民间卦例、课程讲义等材料统一归为“现代案例/现代断法参考”。默认不进入主判据。

调用条件：

- 用户明确要求“参考南山真人风格/现代案例派”。
- 当前问题需要展示象意展开方式。
- 对同一卦做复盘，而非直接给确定结论。

输出时必须标明：

> 此部分为现代案例派/南山真人式象意展开，不等同于默认传统纳甲主判。

不得直接套用历史案例中的具体人物、疾病、地理、职业、外貌、隐情等细节。

## 12. 禁止行为

- 禁止在 fresh 模式读取旧案例。
- 禁止把所有流派混合成一个看似确定的断语。
- 禁止不标来源地使用现代案例断语。
- 禁止用六神、神煞、纳音、象意推翻明确的用神旺衰动变结构。
- 禁止将未校对 OCR 直接作为高权重依据。
- 禁止把古籍断语当成不需要条件的绝对结论。
- 禁止为了迎合用户而只列支持证据，不列反向证据。
- 禁止在缺少起卦时间、月日、动爻时输出高置信度结论。
- 禁止把低置信度象意说成确定事实。
